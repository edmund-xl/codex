# Training Workflow

这份文档定义的是：以后你怎么给材料，我怎么把它训练进系统。

目标不是一次性修一个 bug，而是让 MegaETH AI Security Platform 持续变准。

## 1. 最佳协作方式

每次尽量围绕“一类材料”来训练，而不是把不同类型的文件混在一起。

推荐分类：

- EASM 类
- Endpoint / Incident 类
- Host / Baseline / Risk Analytics 类
- Cloud / IAM / KMS 类
- CI/CD / Repo / Secret / Dependency 类

## 2. 每次训练最好提供什么

### A. 原始材料

至少提供一个真实文件：

- `csv`
- `json`
- `log`
- `txt`
- `md`
- `pdf`

### B. 你认为正确的分类

至少告诉系统：

- `source_type`
- `event_type`
- 应该调用哪些 `skills`

### C. 你认可的分析结论

至少说明：

- 是不是真的有问题
- 风险级别
- 为什么这么判断
- 哪些是误报线索
- 哪些才是关键证据

### D. 你不满意当前系统的地方

例如：

- 分错类了
- skill 分配太简单
- 报告抓错重点
- 风险级别不对
- 报告太像模板，不像分析师

## 3. 推荐输入模板

以后如果你愿意，可以直接按这个格式给我：

```text
文件：
<文件路径>

正确分类：
source_type =
event_type =
skills =

正确结论：
- 是否真实问题：
- 风险级别：
- 关键依据：
- 是否误报：
- 建议动作：

当前系统哪里不对：
- ...
```

## 4. 系统该怎么学习

每一轮训练，我会把你的输入拆成四层修正：

1. 文件识别  
让系统先知道这份材料“本质上是什么”。

2. 分类  
修正 `source_type` 和 `event_type`。

3. 技能规划  
修正应该调用哪些 MegaETH 能力。

4. 报告  
修正：
- 结论方向
- 风险级别
- 证据提炼
- 建议动作
- 输出风格

## 5. 系统里已经支持的学习动作

### 自动学习

成功分析后，系统会自动写入：

- `memory_feedback`
- auto-observation

### 手工确认

在 Intake 里点：

- `这次分类正确，记住它`

### 手工纠偏

在 Intake 里点：

- `这次分类不对，我来纠正`

然后选择：

- 正确 source type
- 正确 event type
- 正确 skills

## 6. 训练优先级建议

如果要让质量提升最快，建议优先训练：

1. 容易误判但经常出现的文件
2. 你最常看的报告类型
3. 误报和真实问题最难区分的材料
4. 你最在意的输出风格

## 7. 一次训练后的理想结果

一轮训练结束后，至少应该看到：

- 相似材料分类更准
- 相似材料 skill 分配更准
- 报告不再跑偏到完全错误的方向
- Memory Rules / Feedback 有更干净的积累

## 8. 长期目标

长期目标不是“把所有规则写死”，而是形成一套：

- 样本驱动分类
- 用户纠偏驱动学习
- 可回归验证
- 可逐步增强报告质量

的安全分析工作流。
